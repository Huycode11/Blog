Java Exception Handling – Xử lý ngoại lệ trong Java

Tác giả: Văn Huy | Ngày đăng: 19/10/2025

1. Giới thiệu về Exception trong Java

Trong quá trình thực thi chương trình, các lỗi hoặc tình huống bất thường có thể xảy ra — chẳng hạn như chia cho 0, truy cập mảng ngoài phạm vi, hoặc kết nối tệp thất bại. Những lỗi này được gọi là ngoại lệ (Exception).

Java cung cấp một cơ chế mạnh mẽ để xử lý ngoại lệ, giúp chương trình tránh bị dừng đột ngột và cho phép người lập trình xử lý lỗi hợp lý.

2. Phân loại Exception

  • Checked Exception: Được kiểm tra tại thời điểm biên dịch (compile-time), ví dụ: IOException, SQLException.
  • Unchecked Exception: Xảy ra trong quá trình chạy (runtime), ví dụ: NullPointerException, ArithmeticException.
  • Error: Lỗi nghiêm trọng mà lập trình viên không thể kiểm soát, ví dụ: OutOfMemoryError.

3. Cấu trúc try – catch – finally


public class DemoTryCatch {
    public static void main(String[] args) {
        try {
            int result = 10 / 0; // lỗi chia cho 0
            System.out.println("Kết quả: " + result);
        } catch (ArithmeticException e) {
            System.out.println("Lỗi: Không thể chia cho 0!");
        } finally {
            System.out.println("Khối finally luôn được thực thi.");
        }
    }
}

Trong ví dụ trên, khi lỗi xảy ra, chương trình không bị dừng đột ngột mà nhảy đến khối catch để xử lý và sau đó thực thi finally.

4. Từ khóa throw và throws

- throw: Dùng để ném (phát sinh) một ngoại lệ cụ thể trong chương trình.
- throws: Khai báo rằng phương thức có thể ném ra ngoại lệ. 


public class TestThrow {
    static void checkAge(int age) throws ArithmeticException {
        if (age < 18) {
            throw new ArithmeticException("Chưa đủ tuổi đăng ký!");
        } else {
            System.out.println("Đăng ký thành công!");
        }
    }

    public static void main(String[] args) {
        checkAge(16);
        System.out.println("Kết thúc chương trình");
    }
}

5. Tạo Exception tùy chỉnh (Custom Exception)

Đôi khi bạn muốn tạo loại ngoại lệ riêng để phù hợp với logic nghiệp vụ. 


class InvalidEmailException extends Exception {
    public InvalidEmailException(String message) {
        super(message);
    }
}

public class EmailChecker {
    static void validate(String email) throws InvalidEmailException {
        if (!email.contains("@")) {
            throw new InvalidEmailException("Email không hợp lệ!");
        } else {
            System.out.println("Email hợp lệ!");
        }
    }

    public static void main(String[] args) {
        try {
            validate("vanhuygmail.com");
        } catch (InvalidEmailException e) {
            System.out.println("Lỗi: " + e.getMessage());
        }
    }
}

6. Kết luận

Cơ chế xử lý ngoại lệ trong Java giúp chương trình trở nên an toàn, dễ bảo trì và ngăn chặn việc dừng đột ngột khi lỗi phát sinh. Việc nắm vững try-catch-finally, throw, throws và tạo Custom Exception là kỹ năng quan trọng trong lập trình Java chuyên nghiệp.
